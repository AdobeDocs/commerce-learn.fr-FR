# Architecture pour GRA

Découvrez comment architecturer l’architecture de référence globale (GRA). Pour mieux préparer une GRA à une implémentation réussie, les experts d’Adobe discutent de ce qui est et de ce qui n’est pas une GRA. Une discussion approfondie sur la gestion du code à l’aide de packages distincts, de Git partagé et de monorepo. D’autres sujets relatifs à la documentation, aux outils et aux attentes du personnel sont abordés afin d’élaborer leurs parties pour une mise en œuvre réussie de la GRA.

## Audience

* Toute personne ayant besoin de comprendre ce concept.
* Projets volumineux avec plusieurs marques

## Contenu vidéo

* Ce qui est considéré et ce qui n’est pas considéré comme une architecture de référence globale
* Différentes techniques de gestion du code : packages distincts, git divisé et monorepo
* Problèmes courants et problèmes
* Attentes en matière de documentation
* Gestion des bogues et logiciels et outils associés
* Opérations de développement, leads et tests automatisés
