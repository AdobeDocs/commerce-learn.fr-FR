# Migration vers Adobe Commerce - Gel du code

L’implémentation d’un gel de code lors de la migration de votre Adobe Commerce est essentielle pour garantir la stabilité et la réussite du lancement. En interrompant les déploiements de nouveau code, vous permettez à l’équipe d’assurance qualité, aux développeurs et aux parties prenantes de travailler avec une base de code statique, ce qui accroît la confiance et réduit le risque de défauts. Ce processus implique des tests et une validation approfondis, la résolution des bogues hautement prioritaires et la garantie du bon fonctionnement de tous les aspects du site. Un gel de code bien exécuté profite à l’ensemble du projet, évite les problèmes de dernière minute et assure une transition en douceur vers Adobe Commerce.


## À qui s&#39;adresse cette vidéo ?

* Les chefs de projet s’efforcent de garantir une exécution fluide du projet et de réduire les risques.
* Les développeurs et développeuses qui cherchent à comprendre la nécessité d’une base de code stable pour les tests finaux.
* Les équipes d’assurance qualité tentent de clarifier le calendrier et les attentes des tests.
* Parties prenantes souhaitant reconnaître l’importance de la validation finale et de la préparation au lancement.

## Contenu vidéo {#video-content}

* Assurer une base de code statique pour les tests d’assurance qualité finaux et la validation afin d’éviter les problèmes de dernière minute.
* Coordination des tests manuels et automatisés, des tests de charge et des tests de pénétration pour garantir que tous les aspects du site fonctionnent comme prévu.
* Impliquer les parties prenantes de l’entreprise, les développeurs et les équipes informatiques dans le processus de validation afin d’assurer un lancement fluide et réussi.
